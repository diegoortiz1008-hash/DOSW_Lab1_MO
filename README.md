# DOSW_Lab1_MO
Primer laboratorio de DOSW
Respuestas cuestionario (10%)
1.merge combina historiales creando un commit de unión; rebase reescribe el historial moviendo commits para que parezca una línea continua.
2.Se produce un conflicto de merge y Git te obliga a resolverlo manualmente antes de completar la fusión.
3.git log --graph --oneline --all --decorate
4.Un commit guarda cambios en tu repositorio local; un push envía esos commits al repositorio remoto.
5.git stash guarda temporalmente cambios sin commit; git pop los restaura y los elimina del stash.
6.HashMap no es sincronizado y permite null; HashTable es sincronizado (thread-safe) y no permite null.
7.Collectors.toMap() es más declarativo, conciso y se integra con streams, reduciendo código y errores manuales.
8.Es una operación de transformación (mapear datos a otra forma).
9.filter() selecciona elementos que cumplen una condición y retorna un Stream filtrado.
10.Cambiar a develop: git checkout develop
  Actualizar: git pull
  Crear rama: git checkout -b feature/nombre
  Trabajar y hacer commits
  Push: git push origin feature/nombre
11.git branch solo crea la rama; git checkout -b la crea y cambia a ella.
12.Permite aislar cambios, evitar romper main, facilitar revisiones y mantener un flujo de trabajo organizado.
